  • 内部命令使用
    help cd

  • 外部命令使用

info

文件管理

文件查看

  • pwd

  • cd
    d - ，进入上一次目录

  • ls

    • -l long长格式，显示详细信息
    • -a all全部文件，显示隐藏文件
    • -r reverse逆向，默认使用文件名逆向
    • -t time时间，按时间顺序显示
    • -R recursion递归，显示子文件夹及子文件信息

    可以组合使用，如：ls -lartR

目录的查看与删除

  • mkdir
    mkdir -p /a/b/c, 创建多级目录
  • rm
    • rmdir 删除空目录
    • rm -r 删除非空目录

复制和移动

  • cp 复制文件和目录

    • -r 复制目录
    • -p 保留用户、权限、时间等文件属性
    • -a 等同与-dpr
    • -v 显示复制过程

  • mv 移动或重命名文件

  • 通配符

    • 星号*，匹配多个字符
    • 问号?，匹配单个字符

文本查看命令

  • cat 文本内容显示到终端
  • head 查看文件开头
  • tail 查看文件结尾，-f同步更新显示
  • wc 统计文件内容信息

VI编辑器

VI/VIM 编辑器的四种模式

  • 正常模式
  • 插入模式
  • 命令模式
  • 可视模式

正常模式

  • 除小箭头外的上下左右移动光标方式

    • H - 左
    • J - 下
    • K - 上
    • L - 右

  • 复制命令

    • yy - 复制光标所在的整行（包括换行）
    • 数字yy - ‘数字’+yy，复制包括光标所在的一下‘数字’个行（包括换行）
    • y$ - 复制从光标位置到行结尾的字符（不包含换行）

  • 剪切命令（用法与复制一样）

    • dd
    • 数字dd
    • d$

  • 粘贴命令

    • p - 粘贴复制的字符（如果有换行，下面行下移）

  • 撤销命令

    • u - 撤销操作，可撤销多次

  • 重做撤销命令

    • Ctrl+R - 重做撤销的一步，可多次重做

  • 删除单个字符命令

    • x - 删除光标所在位置的单个字符

  • 替换单个字符命令

    • r+’替换字符’ - 用‘替换字符’替换光标所在位置的单个字符

  • 移动光标到所在行

    • ‘行号’+G - 即 ‘行号’+Shift+g
    • g - 移动光标到文本第一行
    • G - 移动光标到文本最后一行
    • ^ - 移动光标到所在行的首部
    • ¥ - 移动光标到所在行的尾部

插入模式

从正常模式进入插入模式的几种做法

  • i - 从当前光标插入
  • I - 从当前光标的行首插入
  • a - 从当前光标的后一个位置插入
  • A - 从当前光标的行尾插入
  • o - 从当前光标的下一行插入新行
  • O - 从当前光标的上一行插入 新行

命令模式

  • 保存

    • ：w ‘filename’ - 保存到文件’filename’
      注意，若VI打开时没有加文件名，则必须输入filename，否则无法保存；若VI打开时有文件名，则只输入w默认保存当打开的文件；若VI打开时有文件名，但输入w+另外一个文件名，则想当与另存为

  • 退出

    • ：q - 退出，wq保存退出，q!不保存退出

  • 执行Linux命令

    • ：!+‘Linux命令’，例如：!ifconfig

  • 查找字符

    • ：/+‘字符’，查找到之后光标移动到第一个找到到位置，使用n查看下一个位置，shift+n查看上一个位置

  • 替换字符

    • ：s/旧字符/新字符 - 例如，将f替换F为 ：s/f/F ，但是默认只替换光标所在行的第一个字符，若光标所在行没有该字符，则提示错误
    • ：%s/旧字符/新字符 - 替换所有行的第一个旧字符
    • ：%s/旧字符/新字符/g - g=global,替换全局的旧字符
    • ：起始行,结束行+以上替换方式，即为在指定起始行到指定行之间进行替换操作

  • VIM设置

    • ：set+设置项 - 但该设置当次有效，若需要永久保存设置，需要修改/etc/vimrc 文件

可视模式

用户管理

用户管理命令

  • useradd - 新建用户(仅root用户有权限创建)

    • /etc/passwd

    • /etc/shadow

    • id

  • userdel - 删除用户

    • 加-r才能删除用户的家目录

  • passwd - 修改用户密码

  • usermod - 修改用户属性

    • -g 加入用户组

  • chage - 修改用户属性

组管理命令

  • groupadd - 新建用户组
  • groupdel - 删除用户组

用户切换

  • su - ‘username’ ： 做临时的用户替换，’-‘表示切换用户的同时将运行环境也切换为username的运行环境

  • su 切换用户

  • sudo 以其他用户身份执行命令

    • visudo - 设置需要使用sudo的用户（组）
    • 也可以直接编辑 /etc/sudoers 文件

用户（组）配置文件

  • /etc/passwd文件 - 用户配置文件，保存用户信息，可以手动修改

  • /etc/shadow文件 - 保存用户和用户密码相关信息

  • /etc/group文件 - 保存用户组相关信息

查看文件权限

  • 文件类型

    • ‘-‘ 普通文件
    • ‘d’ 目录文件(directory)
    • ‘b’ 块特殊文件(block)
    • ‘c’ 字符特殊文件(character)
    • ‘l’ 符号链接(link)
    • ‘f’ 命名管道(fifo)
    • ‘s’ 套接字文件(socket)

  • 字符权限表示方法

    • r - 可读（read）
    • w - 可写 (write)
    • x - 可执行 (exec)

  • 数字权限的表示方法(8进制表示法)

    • r = 4
    • w = 2
    • x = 1

创建新文件时的默认权限值根据umask值计算，属主和属组根据当前进程的用户来设定

  • 目录权限的表示方法
    • x - 进入目录
    • rx - 显示目录内的文件名
    • wx - 修改目录内的文件名

      当属主和属组权限冲突时，以属主的权限为准d

修改文件权限

  • chmod - 修改文件权限

    字符修改方式

    • u 修改属主的权限(user)
    • g 修改属组的权限(group)
    • o 修改其他用户的权限(other)
    • a 修改所有的权限(all)

      修改符号： +增加权限，-减少权限，=设置权限

    数字修改方式

    默认权限 = 666-umask

  • chown - 修改文件属主、属组

    • chown username file 修改属主
    • chown :groupname file 修改属组
    • chown username:groupname file 同时修改属主和属组

  • chgrp - 可以单独修改属组，不常用

    • chgrp groupname file 修改属组

注意：权限限制是限制非root用户的，root用户不受限制

网络管理

网络状态查看

  • net-tools

    • ifconfig

      eth0:第一块网卡(网络接口)
      第一个网络接口可能叫一下几个名字
      eno1 板载网卡
      ens33 PCI-E网卡
      enp0s3 无法获取物理信息的PCI-E网卡
      CentOS7使用了一致性网络设备命令，以上都不匹配则使用eth0

  • route
  • netstat
  • iproute2
    • ip
    • ss

网络接口命名修改

  • 网卡命名规则受biosdevname和net.ifnames两个参数影响
  • 编辑/etc/default/grub文件，增加biosdevname=0 net.ifnames=0
  • 更新grub
    • grub2-mkconfig -o /boot/grub2/grub.cfg
  • 重启
    • reboot
biosdevname et.ifnames 网卡名
默认 0 1 ens33
组合1 1 0 em1
组合2 0 0 eth0

查看网络情况

查看网卡物理连接情况
mii-tool eth0

查看网关

    使用-n参数不解析主机名

网络配置命令

  • ifconfig <接口> <IP地址> <netmask 子网掩码> 设置网卡配置
  • ifup <接口> 启动网卡（ifconfig <接口> up）
  • ifdown <接口> 关闭网卡（ifconfig <接口> down）

网关配置命令

  • route add default gw <网关ip>
  • route add -host <指定ip> gw <网关ip>
  • route add -net <指定网段> netmask <子网掩码> gw <网关ip>

进程管理

查看进程

  • ps - 查看进程状态（process state）

    • ps - 查看当前终端下能够查看到的 进程
    • ps -e - 查看其他进程

  • pstree - 查看进程树

  • top - 更全面的查看进程信息

    • top -p + 进程ID

进程是树形结构，进程和权限有着密不可分的关系

进程优先级调整

  • 调整优先级

    • nice 范围从-20 ~ 19，值越小，优先级越高，抢占资源就越多

    • renice 在不关闭进程的情况下重新设置优先级

  • 进程的作业控制

    • 后台运行，&

    • jobs 命令 ：显示当前shell中已经启动的作业状态

    • fg + 编号 ： 将后台对应编号的程序调入前台运行

    • Ctrl+Z ：将前台当前运行的程序挂起

    • bg + 编号 ： 后台对应编号的程序调入后台运行

进程间通信

  • kill -l 显示信号名称，若不加信号编号，则列出全部信号名称

  • kill -9 PID 向PID程序发生编号为9的信号，即SIGKILL，无条件杀死

守护进程（daemon）

  • nohup命令：即执行程序忽略hangup命令，通常和&配合使用

  • screen命令：实现当前窗口与任务分离，在远程操作时，防止因断开连接导致任务被杀死

    • screen 进入screen环境
    • ctrl+a d 退出（detached）screen环境
    • screen -ls 查看screen的会话
    • screen -r + screenid恢复会话

服务管理工具systemctl

  • 服务集中管理工具
    • service
    • systemctl
      • 常见操作:systemctl start|stop|restart|reload|enable|disable 服务名称
      • status 查看状态
      • start 启动服务
      • stop 终止服务
      • restart 重启服务，先停掉再启动，一般用于重新加载配置文件时
      • reload 服务不停止，重新加载配置文件
      • enable 服务随着开机一起运行起来
      • disable 服务不随着开机一起运行起来
