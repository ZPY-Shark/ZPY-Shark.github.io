算法基础-复杂度

发表于 2019-07-15 | 分类于 数据结构与算法
数据结构与算法的基础知识，复杂度概念及计算。

为什么需要复杂度分析？

需要一个不用具体数据来测试，就可以粗略估算算法执行效率的方法。

大O复杂度表示法

规律：在一个循环中，所有代码的执行时间T(n)与每行代码的执行次数成正比。

T(n)=O(f(n))

其中，

  • T(n)表示代码执行的时间；
  • n表示数据规模的大小；
  • f(n)表示每行代码执行的次数总和。
  • O表示代码执行时间T(n)与f(n)成正比。

大O时间复杂度实际并不具体表示代码真正的的执行时间，而是表示代码时间随数据规模增长的变化趋势。所以，也叫渐进时间复杂度，简称时间复杂度。

时间复杂度分析

当n很大时，公式中的 低阶、常量、系数 三部分并不左右增长趋势，所以都可以忽略，只需要记录一个最大量级即可。

  • T(n)=O(2n+2) 等价于 T(n)=O(n)
  • T(n)=O(2n2+2n+3) 等价于 T(n)=O(n2)

复杂度计算要点

  • 只关注执行次数最多的一段代码
  • 加法法则：总复杂度等于量级最大的那段代码的复杂度
  • 乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积

几种常见的时间复杂度

多项式量级

  • 常量阶 O(1)
  • 对数阶 O(logn)
  • 线性阶 O(n)
  • 线性对数阶 O(nlogn)
  • 平方阶、立方阶、k次方阶 O(n2)、O(n3)、O(nk)

非多项式量级

  • 指数阶 O(2n)
  • 阶乘阶 O(n!)

常见的多项式时间复杂度

O(1)

O(logn)、O(nlogn)

2x = n 求解得 x = log2n
不管对数的底为多少，时间复杂度均为O(logn)
因为对数之间是可以相互转换的，log3n=log32 * log2n
忽略log32这个常量系数
所以O(log3n)等于O(log2n)
在对数阶时间复杂度里，忽略对数的底，统一表示为O(logn)

O(nlogn)也是一种常见的时间复杂度，比如归并排序、快速排序

O(m+n)、O(m*n)

代码的复杂度由两个数据的规模决定

由于无法事先评估m和n的量级哪个大，所以不能简单的利用加法原则，因此复杂度为O(m+n)

均摊时间复杂度

  • 最好情况时间复杂度O(1)
  • 最坏情况时间复杂度O(n)
  • 平均情况时间复杂度

摊还分析

均摊时间复杂度就是一种特殊的平均时间复杂度
当大部分情况下，时间复杂度都很低，只有个别情况下时间复杂度很高，且这些操作之间存在前后连贯的时序关系时，可以考虑是否可以将耗时的那次操作平摊到复杂度比较低的操作上

一般情况下，能使用均摊时间复杂度分析的地方，均摊时间复杂度就等于最好情况时间复杂度
