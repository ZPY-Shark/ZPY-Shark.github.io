Shell脚本学习

发表于 2019-08-13 | 分类于 Linux基础
Linux基础 Shell脚本编程知识学习。

什么是Shell

  • Shell是命令解释器，用于解释用户对操作系统的操作
  • Shell有很多，cat /etc/shells
  • CentOS7、Ubuntu默认使用bash

Linux的启动过程

  • BIOS
  • MBR
  • BootLoader（grub）
  • kernel
  • systemd
  • 系统初始化
  • shell

Shell脚本的格式

  • UNIX哲学：一条命令只做一件事
  • 为了组合命令和多次执行，使用脚本文件来保存需要执行的命令
  • 赋予该文件执行权限（chmod u+rx filename）

Shell脚本的组成

  • Sha-Bang
  • 命令
  • #号开头的注释
  • chmod u+x filename可执行权限

Shell脚本的执行方式

  • bash a.sh

    • 可以没有可执行权限

  • ./a.sh

    • 必须要有可执行权限

以上两种方式执行会产生一个新的进程，不会影响当前进程环境

  • source .a.sh

    • 可以没有可执行权限

  • . a.sh

    • 可以没有可执行权限

这两种方式执行不会产生新的子进程，会对当前环境造成影响

内建命令和外建命令的区别

  • 内建命令不需要创建子进程
  • 内建命令对当前Shell生效

管道

  • 管道和信号一样，是进程间通信的一种方式
  • 匿名管道(管道符)是Shell编程经常用到的通信工具
  • 管道符是“|”，将前一个命令的结果传递给后面的命令
    • ps | cat
    • ehco 123 | ps

重定向

  • 一个进程默认会打开 标准输入、标准输出、错误输出 三个文件描述符

  • 输入重定向符号 ‘<’

    • read var < /path/file

  • 输出重定向符号

    • ‘>’ ：清空文件内容后再输入内容
    • ‘>>’：不清空文件内容，在下一行追加输入内容
    • ‘2>’：对文件描述符2的操作，即前面命令产生的错误输入到文件中，错误重定向
    • ‘&>’：无论输出什么信息全部输入到文件中

  • 输入和输出重定向组合使用

变量

变量的定义

  • 变量名的命名规则
    • 字母、数组、下划线
    • 不能以数字开头

变量的赋值

Shell中的变量为弱类型的变量，不区分类型

  • 为变量赋值的过程称为 变量替换

    • 变量名=变量值

      a=123

    • 使用let为变量赋值

      let a=10+2

    • 将命令赋值给变量

      l=ls

    • 将命令结果赋值给变量，使用$()或者``

      letc=$(ls -l /etc)

    • 变量值有空格等特殊字符可以包含在””或’’中，当字符串内包含双引号时，可以用单引号；当字符串内包含单引号时，可以用双引号

    等号两边不得有空格，变量名、变量值中不得包含空格

变量的引用

  • ${变量名} 称作 对变量的引用
  • echo ${变量名} 查看变量的值
  • ${变量名} 在部分情况下可以省略为$变量名

变量的作用范围

  • 变量的默认作用范围

    默认只对当前Shell进程有效

  • 变量的导出

    export

    在定义时加上export关键字，可以在子进程中使用

  • 变量的删除

    unset
