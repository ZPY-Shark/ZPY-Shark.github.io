Shell脚本学习

发表于 2019-08-13 | 分类于 Linux基础
Linux基础 Shell脚本编程知识学习。

什么是Shell

  • Shell是命令解释器，用于解释用户对操作系统的操作
  • Shell有很多，cat /etc/shells
  • CentOS7、Ubuntu默认使用bash

Linux的启动过程

  • BIOS
  • MBR
  • BootLoader（grub）
  • kernel
  • systemd
  • 系统初始化
  • shell

Shell脚本的格式

  • UNIX哲学：一条命令只做一件事
  • 为了组合命令和多次执行，使用脚本文件来保存需要执行的命令
  • 赋予该文件执行权限（chmod u+rx filename）

Shell脚本的组成

  • Sha-Bang
  • 命令
  • #号开头的注释
  • chmod u+x filename可执行权限

Shell脚本的执行方式

  • bash a.sh

    • 可以没有可执行权限

  • ./a.sh

    • 必须要有可执行权限

以上两种方式执行会产生一个新的进程，不会影响当前进程环境

  • source .a.sh

    • 可以没有可执行权限

  • . a.sh

    • 可以没有可执行权限

这两种方式执行不会产生新的子进程，会对当前环境造成影响

内建命令和外建命令的区别

  • 内建命令不需要创建子进程
  • 内建命令对当前Shell生效

管道

  • 管道和信号一样，是进程间通信的一种方式
  • 匿名管道(管道符)是Shell编程经常用到的通信工具
  • 管道符是“|”，将前一个命令的结果传递给后面的命令
    • ps | cat
    • ehco 123 | ps

重定向

  • 一个进程默认会打开 标准输入、标准输出、错误输出 三个文件描述符

  • 输入重定向符号 ‘<’

    • read var < /path/file

  • 输出重定向符号

    • ‘>’ ：清空文件内容后再输入内容
    • ‘>>’：不清空文件内容，在下一行追加输入内容
    • ‘2>’：对文件描述符2的操作，即前面命令产生的错误输入到文件中，错误重定向
    • ‘&>’：无论输出什么信息全部输入到文件中

  • 输入和输出重定向组合使用

变量

变量的定义

  • 变量名的命名规则
    • 字母、数组、下划线
    • 不能以数字开头

变量的赋值

Shell中的变量为弱类型的变量，不区分类型

  • 为变量赋值的过程称为 变量替换

    • 变量名=变量值

      a=123

    • 使用let为变量赋值

      let a=10+2

    • 将命令赋值给变量

      l=ls

    • 将命令结果赋值给变量，使用$()或者``

      let c=$(ls -l /etc)

    • 变量值有空格等特殊字符可以包含在””或’’中，当字符串内包含双引号时，可以用单引号；当字符串内包含单引号时，可以用双引号

    等号两边不得有空格，变量名、变量值中不得包含空格

变量的引用

  • ${变量名} 称作 对变量的引用
  • echo ${变量名} 查看变量的值
  • ${变量名} 在部分情况下可以省略为$变量名

变量的作用范围

  • 变量的默认作用范围

    默认只对当前Shell进程有效

  • 变量的导出

    export

    在定义时加上export关键字，可以在子进程中使用

  • 变量的删除

    unset

系统环境变量

  • 环境变量

    每个Shell打开都可以获得到的变量

    • set 和 env命令

      – env命令查看所有 环境变量

      使用 ‘echo $变量名’ 可以查看变量值

      PATH变量是命令的搜索路径

      向PATH路径添加新路径时使用 PATH=$PATH:/root

      当前定义的PATH环境变量都进行了export导出，所以对当前终端和自己的子Shell生效，对其他Shell不生效

      $PS1，终端显示内容

      – set命令查看所有 环境变量、预定义变量、位置变量

    • 预定义变量

      • $?

        表示上一条命令是否产生了错误，用以确认上条命令是否执行成功

      • $$

        表示当前进程的PID

      • $0

        表示当前进程的名称

    • 系统定义好的位置变量

      位置变量即执行脚本时所带的参数

      $1 $2 … $9 ${10} …

      当参数为空值时，处理参数可能出现问题

环境变量的配置文件

  • /etc/profile
  • /etc/bashrc
  • ~/.bash_profile
  • ~/.bashrc
  • /ect/profile.d/ （目录）

/etc下的配置文件中环境变量对所有用户有效，~/目录下放置特定用户独有的环境变量

  • login shell

    su -username 为login shell

    在用户登录时，四个文件都会执行，加载配置文件比较齐全

    • 执行顺序

      • /etc/profile
      • .bash_profile
      • .bashrc
      • /etc/bashrc

        需要注意执行顺序，避免后面定义的同名变量覆盖掉前面定义的变量

  • nologin shell

    su username 为no login shell

    仅bashrc文件会被执行

    注意nologin shell可能加载环境变量不完全，所以建议使用login shell

    配置文件是每次在用户打开shell终端的时候才会去运行，所以新添加的用户环境变量不会立即生效

    • 让新添加的环境变量生效的方法
      • 退出终端，重新打开
      • 使用source执行使其生效，即source /etc/bashrc

转义、引用、特殊字符

  • 特殊字符

    一个字符不仅有字面意义，还具有元意（meta-meaning）

    • ‘#’ 注释

    • ‘;’ 分号，分隔两行命令

    • ‘\’ 转义符号，

      • 单个字符前的转义符号

        \n \r \t 单个字母的转义

        $ " 单个非字母的转义

  • 引用符号

    常用的引用符号

    • “ 双引号

      当双引号内包含$变量时，为完全引用，显示的是变量的值

    • ‘ 单引号

      当单引号内包含$变量是，为不完全引用，显示的是$变量字符

    • ` 反引号

      反引号内包含执行命令

运算符

  • 赋值运算符（=）

    用于算数赋值和字符串赋值’

    使用unset取消为变量的赋值

    =除了作为赋值运算符还可以作为测试操作符

  • 算数运算符

    • 基本算数运算符（+ - * / ** %）
    • 使用expr命令进行运算，只能支持整数运算，不支持浮点数

  • 数字常量

    • let “变量名=变量值”
    • 变量值使用0开头的八进制
    • 变量值使用0x开头的十六进制

  • 双圆括号

    双圆括号是let命令的简化

    • (( a=10 ))
    • (( a++ ))
    • echo $((10+20))

正常的变量赋值默认为字符串赋值，若需要数字赋值则需要使用 expr / let / 双圆括号 实现

特殊符号大全

  • 引号

    • ‘’ 完全引用
    • “” 不完全引用
    • `` 执行命令

  • 括号

    • 圆括号 () (()) $()

      • 单独使用圆括号会产生一个子shell：(xyz=123)
      • 数组初始化ips=( ip1 ip2 ip3 )
      • ((1+2))用来做算术运算
      • $(ls)执行命令的结果

    • 方括号 [] [ [ ]]

      • 单独使用方括号是测试（test）或数组元素功能
      • 两个方括号表示测试表达式

    • 尖括号 < >

      • ‘<’ 输入重定向
      • ‘>’ 输出重定向

    • 花括号 { }

      • 输出范围 echo {0..9}
      • 文件复制 cp /etc/passwd{,.bak}

  • 运算和逻辑符号

    • 算数运算符 + - * / %
    • 比较运算符 > < =
    • 逻辑运算符 && || !

    0为真（true），1为假（false）

  • 转义符号

    • \转义某字符
    • \n 普通字符转义后有不同的功能
    • ' 特殊字符转义后当做普通字符来使用

  • 其他符号

    • 注释符 #
    • 同一行命令分隔符 ;

      case语句分隔符要使用;;

    • 空指令 :

      永远为真

    • 执行脚本 . 和source命令一样
    • 家目录 ~

      -为上一次目录

    • 分隔目录 ,
    • 通配符 *

      ls * 可以看见当前目录下所有的目录和文件

    • 条件测试或通配符 ?

      ls ?.sh 一个?可以匹配一个字符

    • 取值符号 $
    • 管道符 |
    • 后台运行 &
    • 空格 _

算数比较运算符

  • -gt 大于
